지원 동기 및 목표

소프트웨어 개발자로서 사용자에게 실제로 도움이 되는 서비스를 만드는 것을 가장 중요하게 생각합니다. 빠르게 변화하는 IT 환경에서 새로운 기술을 학습하고 실무에 적용하는 과정 자체를 즐기며, 특히 사용자 경험을 개선하는 기술적 솔루션을 설계하고 구현하는 데 강한 동기가 있습니다.

혁신적인 기술력과 건강한 개발 문화가 자리 잡힌 조직에서 역량을 한 단계 더 성장시키고, 비즈니스와 사용자 모두에게 의미 있는 결과를 만드는 개발자로 기여하고 싶습니다.

성격의 장단점

  • 꾸준한 학습 의지: 온라인 강의와 사이드 프로젝트를 통해 새로운 기술을 습득합니다.
  • 협업과 소통: 개발자, 디자이너, 기획자와 투명한 커뮤니케이션을 중시합니다.
  • 문제 해결 지향: 복잡한 이슈에도 다양한 접근을 시도해 해결책을 도출합니다.

완벽주의 성향으로 일정에 영향을 줄 수 있다는 점을 인지하고 있으며, 우선순위 기준을 명확히 세워 적정 완성도 안에서 성과를 내는 습관을 강화하고 있습니다.

생활신조

"기술은 사람을 위해 존재한다"

기술적으로 뛰어난 결과물도 사용자에게 효용을 주지 못하면 가치가 제한적이라고 생각합니다. 언제나 사용자 관점에서 문제를 정의하고, 실제 사용성을 개선하는 방향으로 개발하고자 합니다.

User-Centered Practical Impact Continuous Improvement

학창시절 경험 및 특이사항

대학교 3학년 교내 해커톤에서 "지역 상권 활성화 앱" 프로젝트로 우수상을 수상했습니다. 팀장으로서 5명의 팀원과 함께 48시간 동안 React Native 기반 앱을 완성했습니다.

백엔드 API 설계부터 프론트엔드 구현까지 전 과정에 참여하며 풀스택 개발 경험을 쌓았고, 제한된 시간 안에서 역할을 효율적으로 분배해 성과를 끌어내는 리더십을 발휘했습니다.

컴퓨터공학과 학회에서 4년간 멘토링을 진행하며 후배들에게 프로그래밍을 지도했고, 복잡한 개념을 구조적으로 설명하는 역량을 키워 팀 내 코드 리뷰와 기술 공유에도 강점을 갖추게 되었습니다.
